# `dd_cli._client` — generated DefectDojo API client

This directory is **vendored, auto-generated code**. Do not edit files here by
hand — your changes will be wiped the next time the client is regenerated.

## Source

Generated by [`openapi-python-client`](https://github.com/openapi-generators/openapi-python-client)
from [`dd-api.json`](../../../dd-api.json) at the repo root, which is the
DefectDojo OpenAPI 3.0.3 spec (currently v2.54.3).

## What's in here

| File / dir | Contents |
|---|---|
| `client.py` | `Client` and `AuthenticatedClient` — HTTP transport built on `httpx`. |
| `errors.py` | `UnexpectedStatus` exception raised on non-2xx responses. |
| `types.py` | `Response`, `UNSET`, `Unset` — sentinel/wrapper types. |
| `api/<resource>/` | One module per endpoint, e.g. `api/products/products_list.py`. |
| `models/` | One module per schema, using [`attrs`](https://www.attrs.org/). |

Endpoint modules expose four functions: `sync`, `sync_detailed`, `asyncio`,
`asyncio_detailed`.

## How to regenerate

When DefectDojo introduces a new endpoint we need, or when we bump
`dd-api.json` to a newer DefectDojo version:

```bash
make install-all     # installs openapi-python-client (one-time)
make generate-client # regenerates this directory
```

Then review the diff carefully — the regen is idempotent given the same input
spec, so any unexpected changes signal an upstream API change worth understanding
before merging.

## Why vendor it?

The plan ([`PLAN.md`](../../../PLAN.md) D17) keeps the generated client
**vendored** rather than built at install time. Trade-offs:

- ✅ `pip install defectdojo-cli` works offline (no network, no codegen tools).
- ✅ Contributors can grep the client directly.
- ✅ CI is faster — no regen step on every build.
- ❌ Larger PRs when we regenerate. Mitigated by isolating regen into its own PR.

## Excluded from quality gates

The generated tree is excluded from:

- `ruff` lint — see `extend-exclude` in `pyproject.toml`.
- `mypy --strict` — see `[tool.mypy.overrides]` for `dd_cli._client.*`.
- coverage reports — see `[tool.coverage.run] omit`.

We rely on `openapi-python-client`'s own type correctness; we type-check the
hand-written wrapper in `dd_cli/client.py` instead (added in M1).
